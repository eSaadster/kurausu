# Feature Illustrations Workflow

**Creates clean, systematic graphics that communicate product benefits at a glance.**

---

## Purpose

Feature sections need visuals that communicate complex capabilities simply. This workflow creates illustrations that make benefits tangible and memorable.

**Use for:**
- Feature cards on landing pages
- Benefit sections
- Product tour graphics
- Capability highlights
- Comparison feature grids
- Pricing page feature lists

---

## Recommended Style

**Primary: Fleet Geometric**
Clean, systematic illustrations with bold shapes and clear iconography.

### Step 1: Define the Feature

**Gather information:**
```
FEATURE NAME: [e.g., "AI-Powered Matching"]
BENEFIT: [What problem does it solve?]
PROOF POINT: [Data that backs it up]
VISUAL CONCEPT: [How to represent it visually]
EMOTIONAL HOOK: [How should viewer feel?]
```

**Example:**
```
FEATURE NAME: AI-Powered Matching
BENEFIT: Finds drivers who actually stay, not just apply
PROOF POINT: 90% first-year retention rate
VISUAL CONCEPT: Puzzle pieces connecting, or magnet attraction
EMOTIONAL HOOK: Relief â€” finally, matches that work
```

---

### Step 2: Choose Visual Approach

**Icon-based (simple features):**
- Single geometric icon
- Clear, recognizable symbol
- Works at small sizes

**Concept illustration (complex features):**
- Abstract representation
- Multiple elements in composition
- Tells a mini-story

**Data-integrated (proof-heavy features):**
- Metric as visual element
- Icon + number combination
- Emphasis on the result

---

### Step 3: Select/Design Icon

**Uptime icon style:**
```
STYLE:
- Geometric, clean lines
- 2-3px stroke weight
- Single color (Dark Blue or Lime)
- No gradients or shadows
- Works at 24px and 64px

SHAPE LANGUAGE:
- Rounded corners (consistent with brand)
- Simple forms (circle, square, line)
- Negative space for clarity
```

### Step 5: Add Uptime Accents

**Accent options:**

1. **Lime icon background**
   - Circle or rounded square behind icon
   - Lime Green #7CFC00 at 15% opacity
   - Icon in Dark Blue

2. **Lime underline**
   - 4px underline under headline
   - Lime Green #7CFC00
   - 50% headline width

3. **Lime border accent**
   - 3px left border
   - Lime Green #7CFC00
   - Full height of card

4. **Lime metric badge**
   - Small pill with proof point
   - Lime background, dark text
   - "90% retention" style

---

### Step 6: Construct Prompt

**Icon + Text Card:**
```
Feature card in Uptime Fleet Geometric style.

CARD:
- Dimensions: 400 x 300px
- Background: White #FFFFFF
- Border radius: 24px
- Padding: 32px
- Subtle shadow: 0 2px 8px rgba(0, 34, 51, 0.08)

ICON:
- Concept: [Describe icon - e.g., "puzzle pieces connecting"]
- Size: 64px
- Style: Geometric outline, 2px stroke
- Color: Dark Blue #002233
- Background: Lime Green #7CFC00 at 15% opacity, circle

HEADLINE:
- Text: "[Feature Name]"
- Font: Inter Bold 20px
- Color: Dark Blue #002233
- Lime underline accent (4px, 40% width)

DESCRIPTION:
- Text: "[1-2 sentence benefit description]"
- Font: Inter Regular 14px
- Color: Dark Blue #002233 at 70%
- Max 2 lines

COMPOSITION: Icon centered top, text centered below, clean hierarchy.

Fleet Geometric style: systematic, clean, professional.
```

**Concept Illustration Card:**
```
Feature illustration in Uptime Fleet Geometric style.

- Thin line or whitespace break
```

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Generic icons | Use icons that relate to trucking/hiring context |
| Inconsistent style | Keep all cards in same visual family |
| Feature-focused copy | Lead with benefit, not feature name |
| No proof point | Add data badge where possible |
| Cluttered cards | One icon, one headline, short description |
| Weak hierarchy | Icon should be most prominent |
| Off-brand colors | Stick to lime, dark blue, white |
| Different card sizes | Keep grid consistent |

---

**Great feature cards make complex capabilities feel simple and inevitable.**
