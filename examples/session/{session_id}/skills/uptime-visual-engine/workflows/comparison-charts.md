# Comparison Charts Workflow

**Creates before/after visuals that make Uptime's advantage undeniable at a glance.**

---

## Purpose

Comparison visuals are Uptime's most persuasive format. They transform abstract improvements into concrete, visual proof that the old way is broken and Uptime fixes it.

**Use for:**
- Before/after sections on landing pages
- "Why Uptime" comparison tables
- Industry benchmark comparisons
- ROI visualization
- Competitive positioning
- Timeline comparisons (84 days â†’ 14 days)
- Cost comparisons

---

## Recommended Style

**Primary: Data Dashboard**
Clean, Metronic-inspired comparison layouts with clear visual hierarchy.

**Secondary: Fleet Geometric**
For more abstract before/after concepts.

### Step 4: Apply Visual Treatment

**"Before" Treatment (Problem/Old):**
```
COLORS:
- Primary: Dark Blue #002233 at 40-50% opacity
- Accent: Muted gray or subtle red for problems
- Background: White or very light gray

STYLE:
- Slightly desaturated
- Can use X marks or warning indicators
- Disconnected elements
- Chaotic or cluttered arrangement

TYPOGRAPHY:
- Inter Medium (not Bold)
- Smaller relative size
- Lower contrast
```

**"After" Treatment (Solution/Uptime):**
```
COLORS:
- Primary: Dark Blue #002233 at 100%
- Accent: Lime Green #7CFC00
- Background: White or Light Green #F6FFED

STYLE:
- Full saturation
- Checkmarks or success indicators
- Connected, flowing elements
- Clean, organized arrangement

TYPOGRAPHY:
- Inter Bold or Black
- Larger relative size
- High contrast
- Lime underline on key metrics
```

### Step 7: Validate

**Checklist:**

- [ ] **Contrast clear?** Old value obviously worse than new
- [ ] **Improvement quantified?** Specific number or percentage shown
- [ ] **Visual hierarchy correct?** New value/Uptime is emphasized
- [ ] **Old value appropriately muted?** Should recede, not compete
- [ ] **Lime accent used?** On the winning/new side only
- [ ] **Message clear in 3 seconds?** Instant understanding
- [ ] **On-brand?** Uptime colors, typography, style
- [ ] **Not mean-spirited?** Factual comparison, not attack

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Both sides equal emphasis | Mute the "before," emphasize the "after" |
| No specific numbers | Always include quantified improvement |
| Lime on both sides | Lime only on Uptime/winning side |
| Attack competitor by name | Compare to "industry" or "traditional" |
| Too many comparisons | One comparison per visual, or themed set |
| Unclear what's being compared | Label both sides clearly |
| Missing the "so what" | Include what the improvement means |
