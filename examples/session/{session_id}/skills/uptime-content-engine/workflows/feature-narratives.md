# Feature Narratives Workflow

**Creates compelling feature descriptions that sell benefits, not features.**

---

## Purpose

Features don't sell‚Äîbenefits do. This workflow transforms technical capabilities into stories that prospects actually care about. Every feature narrative answers: "What problem does this solve, and why should I care?"

**Use for:**
- Landing page feature sections
- Product tour copy
- Feature comparison pages
- Sales enablement content
- App onboarding copy
- Capability highlights

---

## The Uptime Feature Formula

```
FEATURE NARRATIVE = Problem Context + Capability + Outcome + Proof
```

**Example:**
```
PROBLEM CONTEXT: You're spending weeks screening unqualified applicants.
CAPABILITY: Our AI automatically qualifies every applicant against your requirements.
OUTCOME: You only review drivers who actually fit‚Äîsaving hours every day.
PROOF: Customers report 3x more qualified applicants in their pipeline.
```

### Step 1: Define the Feature

**Gather information:**
```
FEATURE NAME: [Internal/technical name]
CAPABILITY: [What it actually does]
PROBLEM IT SOLVES: [What frustration does it eliminate?]
```

### Step 2: Choose Narrative Length

| Context | Length | Elements |
|---------|--------|----------|
| Feature card | Short (25-40 words) | Capability + Outcome |
| Feature section | Medium (50-100 words) | Problem + Capability + Outcome |
| Feature page | Long (150-300 words) | Full narrative with proof |
| Sales deck | Variable | Adjustable per audience |

### Step 3: Write the Problem Context

**Start with the pain. Always.**

**Formula:**
```
[You/Your team/Fleet managers] are [frustrated verb] [painful situation].
[Current approach] [why it fails].
```

**Examples:**
```
SHORT:
"You're losing drivers faster than you can hire them."

MEDIUM:
"Your inbox is flooded with applicants who don't meet basic requirements.
Every day, you waste hours screening resumes that should never have reached you."

LONG:
"Your hiring process wasn't designed for this. Every open position means
weeks of posting to multiple job boards, reviewing hundreds of unqualified
applications, and chasing candidates who ghost you at the last minute.
Meanwhile, your trucks sit idle and your existing drivers burn out
covering the gaps."
```

**Problem context by feature:**
```
AI MATCHING:
"Half your new hires quit within the first year. You're not hiring the
wrong people‚Äîyou're hiring without knowing if they'll actually fit."

AUTOMATED SCREENING:
"For every qualified applicant, you review ten that don't meet
basic requirements. That's hours lost to applicants who were never going to work."

MULTI-CHANNEL SOURCING:
"Your candidates are scattered across five job boards, your website, and
a spreadsheet. Good luck knowing where your best drivers actually came from."

14-DAY GUARANTEE:
"You've been burned before. Fancy promises, zero results. Then the invoice
arrives anyway."
```

### Step 7: Assemble the Narrative

**Short format (feature card):**

### Step 8: Validate

**Checklist:**

- [ ] **Problem first?** Opens with pain, not feature
- [ ] **Specific capability?** Clear what it actually does
- [ ] **Outcome focused?** Describes life after, not just during
- [ ] **Proof included?** Backed by data or testimonial
- [ ] **Uptime voice?** Direct, confident, not salesy
- [ ] **Jargon-free?** Anyone can understand it
- [ ] **Appropriate length?** Matches the context
- [ ] **CTA clear?** (if applicable) Obvious next step

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Feature-first ("We offer...") | Problem-first ("You're struggling with...") |
| Vague capabilities | Specific, concrete descriptions |
| No proof point | Always include data or testimonial |
| Marketing jargon | Plain, direct language |
| No emotional hook | Connect to relief, confidence, or control |
| Too long for context | Match length to placement |

---

**Great feature narratives don't describe what you built. They describe what changes for the customer.**
